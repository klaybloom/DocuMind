# DocuMind RAG 评测清单

这份清单用于内部试用前检查检索和回答质量。每次改检索参数、切分策略、模型或文档解析器后，都建议重新跑一遍。

## 测试数据

为每个知识库准备 5 到 10 个固定问题，覆盖以下类型：

- 精确事实：制度日期、审批时限、负责人、金额上限。
- 编号和术语：合同编号、工单号、产品名、错误码。
- 多片段汇总：需要从同一文档多个段落合并回答。
- 跨文件对比：同一知识库内多个文件的差异。
- 未命中文档：文档没有答案时，助手应明确说明没有找到。
- 知识缺口：未命中文档的问题应出现在知识缺口列表。
- FAQ 草稿：知识缺口应能生成 Markdown FAQ 草稿。
- 负责人：上传时填写负责人后，文件列表应显示负责人。
- 权限和隔离：在 A 知识库提问，不能引用 B 知识库的文件。

## 通过标准

- 检索命中：回答应引用正确文件和片段。
- 引用准确：引用片段能支撑答案，不应只给相似但无关的来源。
- 未命中处理：文档没有答案时，应明确说明未找到，不应编造。
- 缺口记录：未命中问题应被记录，重复问题应增加次数。
- FAQ 草稿：草稿应包含缺口问题、出现次数、最近提问时间和建议负责人。
- 知识库隔离：当前知识库外的文件不能出现在参考来源里。
- 可复现：同一问题重复提问，核心结论和引用应保持一致。

## 建议节奏

- 每次发布前至少跑一组 20 个问题。
- 每新增一个业务知识库，先补 5 个固定问题。
- 每次发现错误答案，把问题加入固定问题集。
